Data Flow B — Review Extraction

What happens when a user opens the Review Panel. Extraction is lazy — it only fires on first open and is cached on the message object for all subsequent reopens (zero API calls).

Diagram 5

Data Flow C — Correction Loop

The full lifecycle of marking an assumption wrong, providing a correction, triggering regeneration, and how the correction persists into every future message in the session. Two Groq API calls per correction: one regeneration + one re-extraction.

flowchart TD A(["User clicks ✗ on an assumption row"]) B["showCorrectionInput = true\nRow expands — textarea + Confirm + Cancel"] C{"User action"} CANCEL(["Input collapses — no state change"]) D["onCorrect propagates up:\nAssumptionRow → AssumptionMap\n→ ReviewPanel → App.handleCorrection"] E["Build AssumptionCorrection\n{ assumption_id, message_id,\noriginal_text, user_correction }"] F["setActiveCorrections append\nCorrections banner updates in ChatView"] G["updateAssumptionStatus marked_wrong\nRow turns red in AssumptionMap"] H["Store message.originalContent\nsave original response text"] I["setRegeneratingMessageId\nMessage dims to opacity-30\nOrange bounce dots appear below"] J["Build correctionNote for ALL active corrections:\nAssume X instead of Y\n— one note per correction in array —"] K["Append correctionNote to\nlast user message in history"] L["groq.chat\nllama-3.3-70b-versatile"] M(["Groq Cloud API"]) N["setMessages: new content\nwas_regenerated = true\nreview_metadata cleared"] O["setRegeneratingMessageId null\nBounce dots disappear\n↺ Regenerated badge shows"] P["See original response\ntoggle becomes available\noriginalContent preserved"] Q["runExtraction auto-fires\nPanel shows skeleton again"] R["New ReviewMetadata\nfresh claims + assumptions\nfor corrected response"] S["Active corrections banner persists\nhowing all corrections as chips\nX to remove individually"] T(["Future handleSubmit calls inject\nall active corrections into SYSTEM_PROMPT\ncorrection memory active for thread"]) A --> B --> C C -->|Cancel / Escape| CANCEL C -->|"Confirm (or Cmd+Enter)"| D D --> E --> F --> G --> H --> I --> J --> K --> L --> M --> N --> O --> P --> Q --> R --> S --> T
